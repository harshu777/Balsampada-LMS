Tuition LMS – Full Project Specification

1. Project Overview
Project Name: Tuition LMS Purpose: Manage online/offline tuition with multi-teacher, multi-class, and multi-subject functionality. Supports multi-step registration, document upload, cash payments, and real-time notifications.
Need Homepage, Contact Page, header, footer. Roles:
* Admin – Manage teachers, students, classes, subjects, payments, approvals, and system settings.
* Teacher – Manage classes, students, content, assignments, tests, and approve student registrations/payments.
* Student – Register, upload documents, enroll in classes/subjects, access learning materials, and track progress.
Tech Stack:
* Backend: Node.js + NestJS (REST + WebSocket support)
* Frontend: React.js + Next.js + Tailwind CSS
* Database: PostgreSQL (structured)
* Cache / Realtime: Redis + Socket.IO
* File Storage: AWS S3 or Cloudflare R2
* Deployment: Docker + Nginx (initially)
* Authentication: JWT + Refresh Tokens

2. Features by Role
2.1 Admin Dashboard
1. Dashboard Overview:
    * Total teachers, students, classes, subjects
    * Pending approvals (teachers/students)
    * Payment status summary
2. Teacher Management:
    * Approve/reject teacher registrations (document view)
    * Assign teachers to classes/subjects
    * Suspend/activate teacher accounts
    * Configure required documents
3. Student Management:
    * Approve/reject student registrations (document view)
    * Assign students to class/subjects
    * Suspend/activate accounts
    * Configure required documents
4. Class & Subject Management:
    * Create/edit/delete classes
    * Add subjects under classes
    * Assign multiple teachers per subject/class
5. Payments (Cash Focus):
    * View pending payments
    * Mark cash payments as received
    * Payment reports (student, class, teacher)
6. Reports & Analytics:
    * Teacher performance reports
    * Student progress reports
    * Cash ledger & payments

2.2 Teacher Dashboard
1. Dashboard Overview:
    * Assigned classes & subjects
    * Pending student approvals (docs + fees)
    * Upcoming classes
2. Profile & Documents:
    * Upload/track documents
    * View approval status
3. Student Management:
    * Approve/reject students
    * Track attendance
    * Communicate via chat/announcements
4. Class & Content Management:
    * Schedule live classes
    * Upload videos, PDFs, notes, assignments
    * Create quizzes/tests
5. Payments (Cash Focus):
    * Approve cash payments from students
    * Payment collection history
6. Reports:
    * Student progress
    * Attendance
    * Cash received

2.3 Student Dashboard
1. Dashboard Overview:
    * Enrolled classes & subjects
    * Upcoming live classes
    * Pending assignments/tests
    * Payment status
2. Profile & Documents:
    * Upload required documents
    * Track approval status
3. Learning Center:
    * Join live classes
    * Watch recorded lectures
    * Download study material
    * Submit assignments
4. Progress & Reports:
    * Subject-wise progress
    * Marks & feedback
    * Certificates (optional)
5. Payments:
    * Track fee payment status
    * Course unlock after approval
6. Communication:
    * Chat with teacher
    * Notifications & announcements

3. Multi-step Registration Flow
Teacher
1. Personal Details
2. Professional Details (qualifications, experience)
3. Documents Upload (controlled by admin)
4. Verification & Approval (admin approves)
5. Dashboard access upon approval
Student
1. Personal Details
2. Class & Subject Selection
3. Documents Upload (controlled by admin & teacher)
4. Payment Approval (cash, teacher/admin marks received)
5. Dashboard access after approval

4. Database Design (High-Level)
Tables
1. Users – id, name, email, role (Admin/Teacher/Student), password, profile info, status
2. Classes – id, name, description
3. Subjects – id, name, class_id
4. TeacherSubjects – teacher_id, subject_id, class_id
5. StudentSubjects – student_id, subject_id, class_id, status, payment_status
6. Documents – user_id, type, file_url, status
7. Assignments – id, subject_id, title, description, due_date
8. Tests/Quizzes – id, subject_id, title, questions
9. Attendance – id, student_id, class_id, date, status
10. Payments – id, student_id, amount, class_id, subject_id, paid_by, approved_by, status
11. Notifications – id, user_id, type, message, read_status
12. Messages/Chat – id, sender_id, receiver_id, message, timestamp
(Can be implemented with Prisma ORM for Node.js/NestJS)

5. Real-time Features
* Chat between teacher & student → Socket.IO
* Live class notifications → WebSocket events
* Approval notifications → instant alert when teacher/admin approves student/documents
* Assignment/test submission notifications
